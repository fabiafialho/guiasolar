Você basicamente precisa avisar o GitHub qual é o seu domínio e depois configurar alguns apontamentos no GoDaddy. Aqui está o passo a passo direto ao ponto: Passo 1: Configurar no GitHub Abra o repositório do seu site no GitHub. Vá em Settings (Configurações) no menu superior. Na barra lateral esquerda, clique em Pages. Role até a seção Custom domain (Domínio personalizado). Digite o seu domínio (ex: seu-dominio.com) e clique em Save. 💡 Nota: Isso vai criar automaticamente um arquivo chamado CNAME na raiz do seu repositório. Passo 2: Configurar no GoDaddy (DNS) Agora você precisa dizer ao GoDaddy para onde enviar as pessoas que acessam seu link. Faça login no GoDaddy e vá até o seu Painel de Controle. Encontre o seu domínio e clique em Gerenciar DNS (ou apenas DNS). Você precisará adicionar ou editar os seguintes registros: 1. Apontar o WWW (Registro CNAME) Procure por um registro do tipo CNAME com o nome www. Se ele já existir, edite-o. Se não, adicione um novo. Tipo: CNAME Nome/Host: www Valor/Destino: seu-usuario.github.io (substitua pelo seu nome de usuário do GitHub). 2. Apontar o domínio principal (Registros A) Para garantir que o seu site abra mesmo se alguém digitar apenas seu-dominio.com (sem o www), você precisa adicionar 4 registros do tipo A. Se já existirem registros do tipo A com o nome @, delete-os ou edite-os para os IPs do GitHub abaixo: Tipo Nome/Host Valor/Destino (IPs do GitHub) A @ 185.199.108.153 A @ 185.199.109.153 A @ 185.199.110.153 A @ 185.199.111.153 Passo 3: Ativar o HTTPS (Segurança) Depois de salvar tudo no GoDaddy, volte para a aba Pages nas configurações do seu GitHub. Pode demorar de alguns minutos até algumas horas para o GoDaddy atualizar os servidores. Assim que o GitHub reconhecer a mudança, a opção Enforce HTTPS ficará disponível para ser marcada. Ative-a para garantir o cadeado de segurança no seu site. Se der algum erro de "DNS check", não entre em pânico! É super normal a internet demorar um pouquinho para espalhar essa nova rota. Só esperar uns 15 minutinhos e tentar atualizar a página do GitHub. CNAME_domainconnect_domainconnect.gd.domaincontrol.com.1 hora esse eu tiro ? Sim, pode remover ou editar esse registro se ele estiver atrapalhando, mas a resposta curta é: você não precisa se preocupar em deletar ele obrigatoriamente. Esse registro _domainconnect é um padrão que a GoDaddy usa para conectar o seu domínio automaticamente a outros serviços (como Microsoft 365, Blogger, etc.). Ele não vai dar conflito com o seu GitHub Pages. O que você realmente precisa garantir para o GitHub funcionar perfeitamente é: Editar o CNAME com o nome www (esse sim deve apontar para seu-usuario.github.io). Deletar ou editar os registros do tipo A antigos com o nome @ (que geralmente apontam para o IP padrão da GoDaddy) e colocar os 4 IPs do GitHub. O que fazer com ele? Se quiser limpar a tela: Pode deletar sem medo. Não vai quebrar o seu site no GitHub. Se preferir deixar: Pode deixar ele lá quieto, ele não interfere no direcionamento do GitHub. Tanto o site guiasolar.online quanto seu nome alternativo estão configurados incorretamente. O domínio não resolve para o servidor do GitHub Pages. Para mais informações, consulte a documentação (NotServedByPagesError). Esse erro NotServedByPagesError acontece quase sempre por dois motivos: ou a GoDaddy ainda está propagando os novos dados pela internet (o que é normal), ou sobrou algum registro antigo da GoDaddy "escondido" que está dando conflito. Vamos resolver isso de forma direta. Siga este checklist rápido na sua conta da GoDaddy: 1. Limpar registros antigos (O principal vilão) Dê uma olhada na sua tabela de registros DNS e limpe o seguinte: Registros Tipo A antigos: Se além dos 4 IPs do GitHub você encontrar qualquer outro registro do tipo A apontando para um IP diferente (como o IP padrão da GoDaddy), delete-o. Registro "Parked": Se houver algum registro com o valor Parked (Estacionado), pode apagar. Sua lista final de registros do tipo A deve ter apenas estes 4: A | @ | 185.199.108.153 A | @ | 185.199.109.153 A | @ | 185.199.110.153 A | @ | 185.199.111.153 2. Validar o CNAME do WWW Certifique-se de que o seu www está assim: CNAME | www | seu-usuario.github.io (substitua pelo seu nome de usuário exato do GitHub, sem o "https://" e sem o nome do repositório). 3. O truque para destravar o GitHub Às vezes o sistema do GitHub fica preso no erro mesmo depois de você consertar tudo na GoDaddy. Para forçar o GitHub a checar o DNS de novo: Vá em Settings > Pages no seu repositório. Apague o texto guiasolar.online do campo Custom domain e clique em Save (isso vai desvincular temporariamente). Aguarde 1 minuto. Digite novamente guiasolar.online e clique em Save. Isso força o GitHub a reavaliar os servidores da GoDaddy imediatamente. Se os registros estiverem certos, o erro deve sumir e a opção de ativar o HTTPS será liberada em instantes.
